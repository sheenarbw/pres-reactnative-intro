Getting started with React Native

...or not

Once upon a time,
I needed a mobile app

  • Needed my app to "wake up" like an alarm clock
  • => Required Native functionality (Android AlarmManager)
  • React + Redux + Sagas = fun fun fun

What is ReactNative?

  • Learn-once run anywhere
  • Native visual layer
  • Executes Js, Js calls native
  • Created by The Facebook
  • initial release: March 26, 2015
  • stable release: 0.61.1 September 25, 2019

ReactNative versus...

ReactNative versus...just native

Android: Kotlin/Java - Apple: Swift/iOS - Fastest runtime - Slowest devtime

ReactNative versus Flutter

Language: Dash - Cross platform - Well loved - Made by Google

ReactNative versus Xamerin

Language: C# - Cross platform - Microsoft - Compiled

ReactNative versus PWAs

Walks like an app - quacks like an app - BUT it's a website

ReactNative versus Ionic

Angular - WebViews are slow - Cordova

Actually getting started

  • Expo
  • react-native-cli

npm install -g expo-cli

expo init expoProject
cd expoProject
npm start

Cons...

  • Expo is slooow (but wouldn't work for my use case anyway I think)
  • react-native-cli: Android studio was a mission to set up
  • Error messages were pretty unclear
  • Missed chrome dev tools a lot
  • Need a device or emulator

And then came LinuxConf

Twist!

Ionic got a re-re-re-remix

Ionic4 === Ionic for everyone

  • Modern web standards: Web Components for the win
  • => Angular / React / Vue / Js
  • Runs in a browser
  • Small files: 6.12 MB (iconic) vs 47 MB (RN)
  • Capacitor instead of Cordova

Getting started with ionic4 + React


npm install ionic -g
ionic start ionicApp --type=react
ionic serve --devapp
